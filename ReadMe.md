#end to end ml project

The initial workflow establishes a robust development framework by creating an isolated Conda virtual environment (venv) with Python 3.10 and initializing version control via Git, complete with a .gitignore to manage local files. To ensure the application is modular, easily distributable, and deployment-ready, the project is structured as a custom Python package. This is achieved by utilizing a setup.py script that dynamically reads dependencies from a requirements.txt file—which includes the -e . flag to trigger a local build—and housing the core logic within a src directory marked by an __init__.py file. Running the standard installation command (pip install -r requirements.txt) simultaneously installs all necessary libraries and compiles the application into a local package, creating a highly organized, industry-standard starting point for continuous development.

In the second phase of the project setup, the focus shifts to establishing a modular, scalable folder structure alongside essential execution-tracking utilities. The core application logic is organized within the src directory, strategically divided into distinct folders such as components (handling data ingestion, transformation, and model training) and pipeline (managing the training and prediction workflows), with standard Python package behavior enforced through __init__.py files. To ensure the codebase remains robust and easily debuggable, custom scripts are implemented for continuous error and event monitoring. A logger.py script is configured to automatically generate timestamped text files within a dedicated logs directory, capturing a permanent timeline of runtime execution details. Concurrently, an exception.py script utilizes Python's sys module to build a custom exception class; whenever the program fails, this class overrides default error outputs to explicitly pinpoint and record the exact script name, line number, and error message. Finally, a utils.py file is initialized to house shared, universally required functions, culminating in a highly organized, industry-standard architecture that is seamlessly pushed to version control.
